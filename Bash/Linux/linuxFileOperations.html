File Operations

File operations in Linux refer to the various actions you can perform on files and directories within the file system. These operations allow users to create, modify, delete, and manage files and directories efficiently.

touch

This command is used to create an empty file or update the timestamp of an existing file. If the file does not exist, touch will create it.

touch

touch flag

  • `-c`: Don't create any files; only change existing files.
  • `-a`: Change the access time of the file.
  • `-m`: Change the modification time of the file.

mkdir

Stands for "make directory." This command creates a new directory (folder) with the specified name. You can create multiple directories at once by listing them after the command.

mkdir

mkdir flag

  • `-p`: Create parent directories as needed.
  • `-m`: Set the permission mode for the new directory.

rm

This command is used to remove (delete) files or directories. It permanently deletes files without moving them to a trash or recycle bin, so caution is advised.

rm

rm flag

  • `-f`: Force deletion without prompting.
  • `-r`: Remove directories and their contents recursively.
  • `-i`: Prompt before every removal.

rmdir

This command removes empty directories. If the directory contains files or other directories, rmdir will not delete it unless the --ignore-fail-on-non-empty option is used (which is not recommended).

rmdir

rmdir flag

  • `-p`: Remove parent directories if they are empty.
  • `--ignore-fail-on-non-empty`: Ignore failure if the directory is not empty.

cp

Stands for "copy." This command copies files or directories from one location to another. You can also use it to create a duplicate of a file in the same directory.

cp

cp flag

  • `-r`: Copy directories recursively.
  • `-i`: Prompt before overwriting files.
  • `-u`: Copy only when the source is newer than the destination.

mv

Stands for "move." This command is used to move files or directories from one location to another. It can also be used to rename files or directories.

mv

mv flag

  • `-i`: Prompt before overwriting files.
  • `-u`: Move only when the source is newer than the destination.
  • `-v`: Show verbose output.
