Introduction to Bash Scripting

Bash (or shell) scripting is a great way to automate repetitive tasks and can save you a ton of time as a developer. Bash scripts execute within a Bash shell interpreter terminal. Any command you can run in your terminal can be run in a Bash script.

There are some conventions to follow to ensure that your computer is able to find and execute your Bash scripts. The beginning of your script file should start with #!/bin/bash on its own line.

The script files also need to have the “execute” permission to allow them to be run. To add this permission to a file with filename: script.sh use:

#!/bin/bash //To ensure this is going to be a bash Script

chmod +x script.sh //Adding permission to be executed

Run the Script in the terminal

./linuxBashScript.sh //After having a script run this command in the terminal

Your terminal runs a file every time it is opened to load its configuration. On Preview: Docs Linux is a family of Unix-like operating systems that are all based on the Linux kernel. Linux style shells, this is ~/.bashrc and on OSX, this is ~/.bash_profile. To ensure that scripts in ~/bin/ are available, you must add this directory to your PATH within your configuration file:

PATH=~/bin:$PATH

Variables

Within bash scripts (or the terminal for that matter), variables are declared by setting the variable name equal to another value.

greeting="Hello" //Note that there is no space between the variable name, the equals sign, or “Hello”

Access Value

To access the value of a variable, we use the variable name prepended with a dollar sign (%).

print the variable value to the screen, we use the following syntax:

echo %greeting

Conditionals

In Bash scripting, conditionals allow you to perform different actions based on whether a condition is true or false. Here are the primary conditional constructs you can use:

// then and fi is the same as { } in Javascript for a if statement
//Note to have space in the bracket
if [ $index -lt 5 ] then //then shows if the condition of the if statement evaluates to true. It comes immediately after the if condition

--- echo $index
else
---echo 5
fi //This keyword marks the end of the if block. It's essentially "if" spelled backward and is used to close the if statement.

Comparison operators

NOTE: (==, !=) are used comparing strings and (-eq, -ne) are used comparing integers.

  • -eq: Equal
  • -ne: Not equal
  • -le: Less than or equal
  • -lt: Less than
  • -ge: Greater than or equal
  • -gt: Greater than
  • -z: Is null

Comparing strings

  • ==: Equal
  • !=: Not Equal

Loops

There are 3 different ways to loop within a bash script: for, while and until.

For

A for loop is used to iterate through a list and execute an action at each step.

for word in $paragraph
do
--- echo $word
done

While

A while loop in Bash is used to repeatedly execute a block of commands as long as a specified condition evaluates to true.

while [ $index -lt 5 ]
do
--- echo $index
--- index=$((index + 1))
done

Until

In Bash, the until loop is similar to the while loop but operates in the opposite manner. It repeatedly executes a block of commands until a specified condition becomes true.

until [ condition ]; do
--- # commands to execute
done
Example:


count=1

until [ $count -gt 5 ]; do
--- echo "Count: $count"
--- count=$((count + 1)) # Increase count by 1
done

echo "Finished!"

Logical Flows

  • while: Executes while the condition is true.
  • until: Executes while the condition is false.

Inputs

To make bash scripts more useful, we need to be able to access data external to the bash script file itself.

read:

#!/bin/bash

//echo = print() and read = input() in python
echo "Please enter your name: " //This prints a message asking for input
read name //: This command waits for the user to type something and press Enter. The input is stored in the variable name.
echo "Hello, $name!"

Aliases

You can set up aliases for your bash scripts within your .bashrc or .bash_profile file to allow calling your scripts without the full filename

alias saycolors='./saycolors.sh'
