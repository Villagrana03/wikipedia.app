sudo

(short for "superuser do") is a command in Unix-like operating systems that allows a permitted user to execute a command as the superuser or another user, as specified by the security policy.

//Basic syntax
sudo [OPTION] COMMAND [ARGUMENTS...]

Common Flags with sudo

-h: Display help information and exit.
-V: Display version information and exit.
-v: Extend the sudo timeout without running a command.

-i: Simulate an initial login, providing an environment similar to a real login session for the specified user.
-s: Run the shell specified in the user's password database entry or the user's shell.

apt

Yes, apt is a command-line tool by itself, specifically designed for package management on Debian-based Linux distributions like Ubuntu. It provides a simplified interface for managing software packages, making it easier to perform various tasks related to installation, upgrading, and removal of packages.

apt commands:

apt update: Update the package index.
apt upgrade: Upgrade all installed packages to their latest versions.
apt install package>: Install a specific package.
apt remove package>: Remove a specific package.
apt purge package>: Remove a package and its configuration files.
apt search keyword>: Search for a package by keyword.
apt list --installed: List all installed packages.
apt show package>: Show detailed information about a specific package.
apt autoremove: Remove packages that were automatically installed to satisfy dependencies but are no longer needed.

apt flags:

-y: Automatically answer "yes" to prompts.
-q: Quiet mode; reduces output.
-s: Simulate the command without making any changes (useful for testing).
--no-install-recommends: Install only the specified package, without recommended packages.
--fix-broken: Attempt to correct a system with broken dependencies.
--allow-downgrades: Allow downgrading of packages.
--allow-remove-essential: Allow removal of essential packages (use with caution).
--reinstall: Reinstall the specified package(s).
