Viewing and Editing Files

Viewing and editing files in Linux involves using commands and text editors to read, modify, or manipulate the contents of files. This is essential for managing configuration files, scripts, and other text-based data.

cat

The cat command is used to concatenate and display the contents of files. It can read one or more files and output their contents to the terminal. It's often used for quickly viewing small files.

cat

cat Flags

  • `-n`: Number all output lines.
  • `-E`: Display a dollar sign at the end of each line.
  • `-A`: Show all characters including hidden ones.

more

The more command allows users to view the contents of a file one screen at a time. It pauses after each screenful of text, enabling users to read large files without overwhelming the terminal.

more

more Flags

  • `-d`: Display a prompt at the bottom of the screen for more input.
  • `-c`: Clear the screen before displaying the next page.
  • `-s`: Squeeze multiple blank lines into one.

less

the less command provides a way to view files page by page. However, it offers more features, including the ability to scroll backward and forward through the file, making it more flexible for navigation.

less

less Flags

  • `-N`: Show line numbers.
  • `-S`: Suppress long lines (horizontal scroll).
  • `-F`: Automatically exit if the content fits on one screen.

nano

nano is a simple, user-friendly text editor for the terminal. It allows users to create and edit text files easily, featuring on-screen commands for saving, searching, and exiting, making it accessible for beginners.

nano

nano Flags

  • `-w`: Disable line wrapping.
  • `-l`: Enable line numbers.
  • `-h`: Display help when editing.

vi

vi is a powerful text editor available in most Unix systems. It operates in different modes (insert, command, etc.) and offers extensive features for editing, but has a steeper learning curve than simpler editors like nano. It's widely used for editing configuration files and scripts.

vim

vi Flags

  • `-R`: Open a file in read-only mode.
  • `-M`: Open in a mode that prevents accidental changes.
  • `-s`: Start in silent mode (no messages displayed).
  • `-c command>`: Execute a command after loading the file.
  • `-u file>`: Use an alternative vimrc file instead of the default one.

tee

Reads from standard input and writes to standard output and one or more files simultaneously.

tee

tee flags

  • -a : Append output to the files instead of overwriting.
  • -i : Ignore interrupts (e.g., signals like Ctrl+C).

cut

Extracts specific sections of each line in a file or input stream, usually by specifying delimiters.

cut

cut flags

  • -d : Define a delimiter to split fields (e.g., `,`, `:`).
  • -f : Select specific fields from the input.
  • -c : Extract specific character positions from each line.

split

Splits files into smaller parts based on line count, byte size, or other criteria.

split

split flags

  • -l : Specify the number of lines per output file.
  • -b : Split by byte size instead of lines.
  • -d : Use numeric suffixes for output files.

tr

Translates, squeezes, or deletes characters from input.

tr

tr flags

  • -d : Delete specified characters from input.
  • -s : Replace repeated characters with a single instance.
  • -c : Complement the set of characters (apply operations to all except specified characters).

sed

Stream editor for filtering and transforming text.

sed

sed flags

  • -i : Edit files in place.
  • -e : Allow multiple editing commands in a single command.
  • -n : Suppress automatic printing, displaying only specified lines.

awk

Pattern scanning and processing language for complex text manipulation.

awk

awk flags

  • -F : Specify a field separator.
  • -v : Pass a variable into the awk program.
  • -f : Specify a file containing awk commands.

head

Displays the first few lines of a file, typically used to preview the beginning of files.

head

head flags

  • -n : Specify the number of lines to display.
  • -c : Display a specific number of bytes instead of lines.
  • -q : Suppress file name headers when outputting multiple files.

tail

Displays the last few lines of a file, often used to monitor the end of files as they grow.

tail

tail flags

  • -n : Specify the number of lines to display.
  • -f : Follow the file as it grows, updating the output in real time.
  • -c : Display a specific number of bytes from the end of the file.
