Nginx

Installing Nginx

To install Nginx on your Raspberry Pi or Ubuntu machine:

sudo apt update

  • Updates your package list.

sudo apt install nginx

  • Installs the Nginx web server.

Checking Nginx Status

To check the status of the Nginx service:

sudo systemctl status nginx

  • Shows if Nginx is active or not, and provides a summary of the service's status.

Starting Nginx

To start the Nginx service:

sudo systemctl start nginx

  • Starts the Nginx service.

Enabling Nginx to Start on Boot

To ensure Nginx starts automatically when the system boots:

sudo systemctl enable nginx

  • Configures Nginx to start automatically when the system starts.

Stopping Nginx

To stop the Nginx service:

sudo systemctl stop nginx

  • Stops the Nginx service.

Restarting Nginx

If you make changes to your configuration, you need to restart Nginx to apply them:

sudo systemctl restart nginx

  • Restarts the Nginx service, which is useful after making changes to config files.

Testing Nginx Configuration

To test if the Nginx configuration is correct (this is useful before restarting):

sudo nginx -t

  • Tests the syntax of the Nginx configuration files.

Viewing Nginx Logs

To view the Nginx access and error logs:

sudo tail -f /var/log/nginx/access.log

  • Shows access logs of requests made to your server.

sudo tail -f /var/log/nginx/error.log

  • Shows error logs for troubleshooting.

Opening Ports in Firewall

If you're using UFW (Uncomplicated Firewall), allow traffic to your desired ports (e.g., port 8080):

sudo ufw allow 8080

  • Allows inbound traffic on port 8080 for your first project.

sudo ufw allow 8002

  • Allows inbound traffic on port 8002 for your second project.

Checking the UFW Status

To check the status of the firewall:

sudo ufw status

  • Shows whether the firewall is active and which ports are open.

  • The `server { ... }` block defines the configuration for handling a particular domain or port.

  • The `listen 8080;` tells Nginx to listen for traffic on port 8080.

  • The `server_name localhost;` specifies the domain or hostname (in this case, `localhost`).

  • The `root /var/www/html/proyect2;` specifies the directory where your project files are located.

  • The `index index.html;` defines the default file to serve when visiting the site (index.html).

Editing the Hosts File

If you want to use custom domains (e.g., `project2.local`), add them to your `/etc/hosts` file:

sudo nano /etc/hosts

  • Add entries like `127.0.0.1 project2.local` to map domain names to your local machine.
