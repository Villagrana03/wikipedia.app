Where

he WHERE clause in SQL is used to filter records based on specified conditions. It limits the data returned by a SELECT query to only those rows that meet certain criteria, such as matching a specific value or range.

Using Where

Filters records based on specified conditions.

SELECT * FROM employees WHERE department = 'Sales';

Filtering with Comparison Operators

  • =: Equal to
  • <> or !=: Not equal to
  • <: Less than
  • <=: Less than or equal to
  • >: Greater than
  • >=: Greater than or equal to

SELECT * FROM orders WHERE amount > 100;

BETWEEN

Filters results within a specified range (inclusive)

SELECT * FROM ProductsWHERE price BETWEEN 10 AND 50;

IN

Matches any value in a specified list.

SELECT * FROM customers WHERE City IN ('New York', 'Los Angeles');

LIKE

Searches for a specified pattern in a column. Use % as a wildcard for any number of characters, _ for a single character.

SELECT * FROM employees WHERE last_name LIKE 'S%';

NULL

IS NULL / IS NOT NULL: Checks for NULL or non-NULL values.

SELECT * FROM orders WHERE delivery_date IS NULL;

Combining Filters with AND, OR, and NOT

  • AND: Combines conditions that must all be true.
  • OR: Combines conditions where any can be true.
  • NOT: Negates a condition.

SELECT * FROM productsWHERE stock > 0 AND price < 20;

SELECT * FROM customers WHERE city = 'Seattle' OR city = 'Portland';

SELECT * FROM employees WHERE NOT department = 'HR';

ORDER BY

Sorts results by one or more columns. Use ASC for ascending (default) or DESC for descending.

SELECT * FROM Products ORDER BY price DESC;

LIMIT

Restricts the number of records returned.

SELECT * FROM orders WHERE amount > 100 LIMIT 10;

Group Filtering with GROUP BY and HAVING

  • GROUP BY: Groups records based on one or more columns.
  • HAVING: Filters groups based on aggregate conditions.

SELECT department, COUNT(*) AS employee_count
FROM employees
GROUP BY department
HAVING employee_count > 5;
