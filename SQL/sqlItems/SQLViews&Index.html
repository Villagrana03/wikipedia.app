Views & Indexes

Views

A View is a virtual table that provides a specific representation of data from one or more tables. It doesn’t store data itself; instead, it runs a query whenever it’s accessed. Views are helpful when you need to simplify complex queries or control access to certain data.

Benefits of Views

  • Simplify Queries: Instead of writing complex queries repeatedly, a view can encapsulate these queries and be called directly.
  • Enhanced Security: Views can limit data visibility by selecting specific columns or rows. For example, you can create a view that omits sensitive information.
  • Data Abstraction: You can expose only necessary data structures while hiding more complex underlying table structures.

Index

An Index is a database object that speeds up data retrieval by creating a quick lookup reference for one or more columns in a table. Like the index in a book, it allows the database to locate rows faster. However, indexes require additional storage and may slow down INSERT, UPDATE, or DELETE operations slightly due to the need to maintain the index.

  • Faster Data Retrieval: Especially for large tables, indexes significantly reduce the time it takes to query data.
  • Improved Query Performance: Queries with WHERE, JOIN, ORDER BY, and GROUP BY clauses benefit from indexes.
