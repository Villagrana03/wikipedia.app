input onChange

When the user interacts with an input, such as entering or deleting any characters, the onChange event fires and updates the component state

onChange: Local Update: When onChange is triggered, it updates the local state of the component (like setting the value of the input field). This is happening within the browser, not on the server.

Server Communication: For data to be sent to the server, you need to explicitly make a request to the server (using something like fetch(), axios, or a form submission). The onChange event doesn't inherently trigger server-side communication.

Controlled vs Uncontrolled

Uncontrolled component: is a component that maintains its own internal state.

Controlled component: is a component that does not maintain any internal state. Since a controlled component has no state, it must be controlled by someone else.

Example of Uncontrolled

Think of a typical element. It appears on the screen as a text box. If you need to know what text is currently in the box, then you can ask the element

let input = document.querySelector('input[type="text"]');

let typedText = input.value; // input.value will be equal to whatever text is currently in the text box.

Example of Controlled

A controlled component, on the other hand, has no memory. If you ask it for information about itself, then it will have to get that information through props. Most React components are controlled.

In React, when you give an element a value attribute, then something strange happens: the input element becomes controlled. It stops using its internal storage. This is a more “React” way of doing things
