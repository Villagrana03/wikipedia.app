Components Interact

A React application can contain dozens, or even hundreds, of components. Each component might be small and relatively unremarkable on its own. When combined, however, they can form enormous, fantastically complex ecosystems of information.

A goood example of a Component is a post in a web, from user data, post data, reaction such as (likes, comments shares etc)

Returning Another Component

As we’ve seen before, each React component is responsible for one piece of the interface. As the application grows in complexity, components need to be able to interact with each other to support the features needed

//you can have components interact with each other by passing information or even returning other components.

import React from 'react';

//We have this Component that is just an image
function Picture() {
return <img src="https://content.codecademy.com/courses/React/react_photo-octopus.jpg" />;
}

//in the following Component Profile(), will have its own data function Profile() {
return (
<>
<Picture /> //here it called Picture component
<h1>Name: Octavia</h1>
<h2>Species: Octopus</h2>
<h2>Class: Cephalopoda</h2>

)
}

export default Profile;

Apply a Component in a Render Function

In the previous lessons, when we defined components and exported them. We usually exported them into our top-level file, App.js. Inside App.js, we imported components and returned them inside our App component—which are then exported to be rendered!

//This is ProfilePage.js import React from 'react';
import NavBar from './NavBar'; //We just imported the NavBar Component from NavBar

function ProfilePage() {
return (
<div>
<NavBar /> //We rendered NavBar (The one we imported already)
<h1>All About Me!</h1>
<p>I like movies and blah blah blah blah blah</p>
<img src="https://content.codecademy.com/courses/React/react_photo-monkeyselfie.jpg" />
</div>
);
}

export default ProfilePage;
