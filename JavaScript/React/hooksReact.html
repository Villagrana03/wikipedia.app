Hooks!

What should we do if we want to add a state to our function component? How about if we wanted our app to respond to the changes in data?

a hook is a special function that lets you use state and other React features in functional components.

Why to use a hook?

React Hooks, plainly put, are functions that let us manage the internal state of components and handle post-rendering side effects directly from our function components

React offers a number of built-in Hooks. A few of these include useState(), useEffect(), useContext(), useReducer(), and useRef().

State Hook

Let’s get started with the State Hook, the most common Hook used for building React components. The State Hook is a named export from the React library, so we import with object destructuring like the following:

import React, { useState } from 'react';

When we call the useState() function, it returns an array with two values:

The current state: The current value of this state.
The state setter: A function that we can use to update the value of this state.

//
const [currentState, setCurrentState] = useState(); //If we can to set a inital value to something, we pass put in inside od the useState()
//If we don’t pass an initial value when calling useState(), the current value of the state during the first render will be undefined. Set From Previous State

React state updates are asynchronous. This means that there are some scenarios where portions of your code will run before the state is finished updating.

it is best practice to update a state with a callback function

// In this example we have 2 callbacks, everytime the user clicks the button, the number will be add one or subtract one

import React, { useState } from 'react';

export default function QuizNavBar({ questions }) {
const [questionIndex, setQuestionIndex] = useState(0);

const goBack = () => //eventHandler
setQuestionIndex((prevQuestionIndex) => prevQuestionIndex - 1); //callback
const goToNext = () =>//eventHandler
setQuestionIndex((prevQuestionIndex) => prevQuestionIndex + 1); //callback

const onFirstQuestion = questionIndex === 0;
const onLastQuestion = questionIndex === questions.length - 1;

return (
<nav>
Question #{questionIndex + 1}
<div>
<button onClick={goBack} disabled={onFirstQuestion}>
Go Back
</button>
<button onClick={goToNext} disabled={onLastQuestion}>
Next Question
</button>
</div>
</nav>
);
}

Arrays in State

JavaScript arrays are the best data model for managing and rendering JSX lists. Let’s take a look at the code for a website for a pizza restaurant.

selected: The current state value.
setSelected: A function to update the state.
initialValue or []: The value that the state will start with. It can be of any type (string, number, array, object, etc.).

export default function PersonalPizza() {
const [selected, setSelected] = useState([]); //This means that selected starts as an empty array. You might use this for managing a list of items, like selected options in a dropdown, checkboxes, or any collection of values.


const toggleTopping = ({target}) => {
const clickedTopping = target.value;
setSelected((prev) => {

// check if clicked topping is already selected
if (prev.includes(clickedTopping)) {
// filter the clicked topping out of state
return prev.filter(t => t !== clickedTopping);
} else {

// add the clicked topping to our state
return [clickedTopping, ...prev];
}
});
};

<div>
{options.map(option => (

{selected.includes(option) ? 'Remove ' : 'Add '}
{option}
</button>
))}
<p>Order a {selected.join(', ')} pizza</p>
</div>
);
}
