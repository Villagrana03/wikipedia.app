Access a Component's props

A component’s props is an object. It holds information about that component.

In React, a "prop" (short for "property") is a special keyword used to pass data from one component to another. Props are how you can customize components and make them reusable.

A good example of why would i use prop, is because whenever we are calling each component in the main app, we can just modify which type of data to render in the prop

function Greeting(props) { //Props are passed as parameters to a React component to allow that component to receive data and behavior from its parent component. By defining props as parameters, you enable components to be flexible and reusable, as they can operate based on the values they receive.
return <h1>Hello, {props.name}! </h1>;
}

Render a Component's props

Props allow us to customize the component by passing it information.

//To make sure that a function Component can use the props object, define your function component with props as the parameter
function Button(props) {
return <button>{props.displayText} </button>;
}


//

To make sure that a function Component can use the props object, define your function component with props as the parameter:


function Button({displayText}) { return <button>{displayText} </button>; }

Here is a example of how props would work

Pass props From Component To Component

Props in React travel in a one-way direction, from the top to bottom, parent to child.

//In this example, App is the parent and Product is the child. App passes three props to Product (name, price, and rating), which can then be read inside the child component
function App() {
return <Product name="Apple Watch" price = {399} rating = "4.5/5.0" />;
}

Render Different UI Based on props

You can do more with props than just display them. You can also use props to make decisions.

//Here is an example of how we can use props to make decisions, if the password is equal to the password another page will render

if (props.password === 'a-tough-password') {
return <h2>Sign In Successful.</h2>
} else {
return <h2>Sign In Failed..</h2>

}

Put an Event Handler in a Function Component

You can, and often will, pass functions as props. It is especially common to pass event handler functions.

//to make a Event Handler we have define it first, We define an event handler as a method on the function component!

import React from 'react';

function Example() {

//Inside of this function handleEvent, we will have an alert
function handleEvent() {
alert(`I am an event handler.
If you see this message,
then I have been called.`);
}

return (
<h1 onClick={handleEvent}> //Everytime we click the <h1> we will call handleEvent function
Hello world
</h1>
);
}

Giving Default Values to props

If nobody passes any text to Button, then Button‘s display will be blank. It would be better if Button could display a default message instead.

import React from 'react';

function Button(props) {
const {text = 'Default Text of Big Button'} = props; //In case in the main app, we do not render something in the ext prop, this will be a default value
return (
<button>{text}</button>
);
}

export default Button;
