Flask Request Object

n Flask, the request object is an instance of the Request class and is used to handle incoming HTTP requests. It contains all the data that was sent by the client (such as a web browser) to the server, and provides methods for retrieving the data in different formats, like form data, URL parameters, JSON, and more.

  • GET: Retrieves data from the server. It is the most commonly used method and should not modify the server state. It is idempotent, meaning repeated requests will produce the same result.
  • POST: Sends data to the server to create or update a resource. It is used when submitting form data, file uploads, or any operation that causes a change on the server. POST is not idempotent.
  • PUT: Sends data to the server to update an existing resource. Unlike POST, PUT is idempotent—repeated requests with the same data will not produce different results.
  • DELETE: Removes a resource from the server. It is idempotent, meaning the same DELETE request can be made multiple times without additional side effects.
  • PATCH: Partially updates a resource on the server. It is not necessarily idempotent, depending on the operation performed.
  • HEAD: Similar to GET, but it only retrieves the headers, not the actual content. It is useful for checking if a resource exists or getting metadata.
  • OPTIONS: Requests information about the communication options available for a specific resource, such as allowed methods.
  • TRACE: Used for diagnostic purposes to track how a request reaches the server, typically for debugging.

Routes Selection

refers to how the Flask application determines which function should handle a specific request from a client (e.g., a browser or API client) based on the URL path and HTTP method (e.g., GET, POST).
