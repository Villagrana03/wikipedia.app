Using Jinga2

Jinja2 is a popular templating engine for Python, commonly used with web frameworks like Flask. It allows developers to create dynamic HTML pages by embedding Python-like expressions and logic directly into HTML templates.

Variable Filters

variable filters are used to modify the value of variables before rendering them in the template. Filters allow you to apply common transformations, formatting, or processing to variables dynamically within your HTML templates.

  • title: Capitalizes the first letter of each word in a string
  • capitalize: Capitalizes the first letter of a string
  • lower: Makes all the characters in a string lowercase
  • upper: Makes all the characters in a string uppercase
  • int: Converts a variable to an integer
  • float: Converts a variable to a floating-point number
  • default: Provides a default value if a variable is not defined
  • length: Calculates the length of a string, list, or dictionary
  • dictsort: Sorts a dictionary by its keys

if statement

if statement is used to conditionally render parts of a template based on a given condition. It allows you to execute a block of code (HTML, for example) only if the specified condition evaluates to true.

{% if condition %}
< p>This text will output if condition is True< /p> Code to render if the condition is True -- >
{% elif another_condition %}
Code to render if the second condition is True -- >
{% else %}
Code to render if no conditions are True -- >
{% endif %}

for loops

you can use for loops to iterate over a list or dictionary and dynamically generate HTML content based on the data passed to the template

{% for item in collection %}
Code to render for each item in the collection -->
{% endfor %}

Inheritance

Template inheritance in Jinja2 allows you to define a "base" template that contains the common structure of your pages, and then extend or modify it in other templates. This is especially useful when you're working on a web application with a lot of pages that share common elements like headers, footers, and navigation.

// base.html (parent)
< div>
< h2>navbar< /h2>
< p>item< /p>
< /div>

{% block content %} #Everything that we extend will show inside of here

{% endblock %}

// index.html (child)
{% extends "base.html" %}
{% block content %}
#inside of here will be all the items that we will be extending!
{% endblock %}
