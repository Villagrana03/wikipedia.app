Templates and Jinja2

Jinja2 is a templating engine for Python, commonly used with web frameworks like Flask. It allows you to write HTML with dynamic content by embedding Python-like expressions inside your HTML files. This makes it easy to generate dynamic pages on the server side.

Using templates to separate HTML from Python code

Templates in Flask allow you to separate the HTML structure from the Python logic. Instead of writing your HTML directly inside your Python code (which is hard to maintain), you write the structure in a separate HTML file (template), and pass dynamic data to it.

@app.route("/")
def index():
user_name = "Michelle" #Lets say this function gets data from a database
--- return render_template("index.html" ,name = user_name)


#index.html
< body>
--- < h1>Welcome, {{ name }}!< /h1> #In this example, {{ name }} is a placeholder for the name variable passed from the Python code.
< /body>

Template inheritance (base templates)

Template inheritance allows you to create base templates that other templates can inherit from, which helps reduce code repetition. You define a base layout (e.g., a common header, footer, or navigation bar), and then other templates can inherit from it and only override specific parts as needed.

#base.html

< header>
--- < h1>Welcome to My Website< /h1>
< /header>

< main>
--- {% block content %}
------ < !-- Content will go here in child templates -->
--- {% endblock %}
< /main>


#home.html

{% extends "base.html" %} This will extend everything from base.html

{% block content %} # every data that we insert inside of this tags will be inside of the template
--- < h2>This is the homepage< /h2>
--- < p>Welcome to the home page of the site!< /p>
{% endblock %}

Passing variables from Flask to templates

In Flask, you can pass variables (like strings, lists, dictionaries, etc.) from your Python code to the template using render_template().

#app.py
@app.route("/")
def home():
user = "Michelle"
items = ["Apple", "Banana", "Cherry"]
--- return render_template("home.html", user_name=user, fruit_list=items )


< home.html
< h1>Welcome, {{ user_name }}!< /h1> # {{ user_name }} will display Michelle on the page.
< ul>
--- {% for fruit in fruit_list %} The for loop will iterate through fruit_list and display each fruit in an unordered list.
------ < li>{{ fruit }}< /li>
--- {% endfor %}
< /ul>

Control structures (loops, conditionals) in templates

Jinja2 allows you to use basic control structures like loops and conditionals inside your templates. This allows you to dynamically generate content based on data passed to the template.

#In the previous example we used a for loop using templates, but now i will show an IF statement

{% if user_name == "Alice" %}
--- < p>Hello, Alice! Welcome back!
{% else %}
--- < p>Hello, guest!< /p>
{% endif %}

Combining control structures:

You can combine loops and conditionals together. For instance, showing a special message for certain users:

< ul>
--- {% for user in users %}
------ {% if user == "Alice" %}
--------- < li>{{ user }} (Special User)< /li>
------ {% else %}
--------- < li>{{ user }}< /li>
------ {% endif %}
--- {% endfor %}
< /ul>
