Introduction to flask

Flask is a popular Python framework for developing web applications. It comes with minimal built-in functionalities and requirements, making it simple to get started and flexible to use. The developer has the liberty to customize most aspects of the app, including database integration, accounts and authentication, and more.

  • Client: The device or browser that a user uses to interact with the application.
  • Web Server: A server that handles HTTP requests from the client and sends back the corresponding responses, like HTML files.
  • Application Server: Processes the business logic of the application. It works behind the web server to handle dynamic content and interact with databases.
  • Web API: A set of endpoints that allow communication between the client and the application server, typically using HTTP methods like GET and POST.
  • Database: Stores and retrieves data that the application uses, such as user information, product details, or transactions.

Creating a Flask App Object

  • app: is the object representing your Flask application. You use it to define routes, configurations, and handle requests.
  • Flask: This is the main class in Flask. It provides all the functionality for handling HTTP requests, defining routes, managing templates, and more.
  • __name__: It tells Flask where to find the app. Flask uses this to locate resources like templates and static files and to set the app's configuration.

# Import Flask class
from flask import Flask


# Create Flask object
app = Flask(__name__)

Creating a route

a route is a URL pattern that is linked to a specific function (called a view function) that is executed when that URL is requested by the client (like a browser). Essentially, routes define the web addresses your application will respond to

@app.route('/')
---def hello_world():
--- return 'Hello, World!' # Output: The text `Hello, World!` is displayed at the URL path '/'

Variable Rules

Variable rules allow a Flask app to respond to dynamic URLs. Variable sections of a URL can be indicated by angular brackets and an optional converter: < converter:variable_name >. These variable parts will then be passed to the view function as arguments.

@app.route(''/page/< int:pg_num >'')
---def content(pg_num):
--- return f'< h1 > Displaying results for page {pg_num}< /h1>'

Render HTML

It can also return HTML to be rendered on a webpage:

@app.route('/')
---def hello_world():
--- return "< h1 >Hello, World!< /h1 >"
