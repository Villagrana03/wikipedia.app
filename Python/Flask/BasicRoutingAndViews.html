Basic Routing and Views

A route in Flask is a URL pattern that is mapped to a Python function (view function). When a user visits a specific URL, Flask will call the function associated with that route and return a response.

Understanding Routes and URL Patterns

@app.route("/") #This is the route
def home():
--- return "< h2 > Home page < /h2 >" # if you render this page at "/" route it wil display that title

@app.route("/about") #This is the route
def About():
--- return "< h2 > About page < /h2 >" #This other route will render the tags at /about route

Handling Methods

  • GET - Used to retrieve data from the server (e.g., loading a page, requesting a resource).
  • POST - Used to send data to the server (e.g., submitting a form, creating a new resource).
  • PUT - Used to update an existing resource on the server (e.g., updating a user profile).
  • DELETE - Used to delete a resource on the server (e.g., removing a post or record).
  • PATCH - Used to make partial updates to a resource (e.g., updating only one field of a form).
  • HEAD - Similar to GET, but it only retrieves the headers of a resource, not the body (useful for checking if a resource exists).
  • OPTIONS - Used to describe the communication options for the target resource (e.g., checking allowed methods for a route).
  • TRACE - Used to perform a message loop-back test along the path to the target resource (less commonly used).

render_templates

render_template is a function in Flask that is used to render an HTML template and return it as the response to the client (usually a web browser). It allows you to separate the HTML content from your Python code by using templates, making your code cleaner and more maintainable.

from flask import Flask, render_template #We need to import render_template

@app.route("/")
def home():
--- return render_template( index.html ) # This will render index.html into the route "/"
