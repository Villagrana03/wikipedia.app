Flask Extensions

Flask is a micro-framework, meaning it provides the basics but doesn't force a lot of extra features. However, you can add extra functionality via Flask Extensions. Some of the most useful ones for building a robust Flask app are:

  • Flask-SQLAlchemy for working with databases (SQL).
  • Flask-WTF for handling forms easily and securely.
  • Flask-Login for managing user authentication.

Flask-SQLAlchemy

Flask-SQLAlchemy is an extension for Flask that adds support for SQLAlchemy, a powerful ORM (Object Relational Mapper) for Python.

from flask imports Flask
// This imports flask from flask_sqlalchemy imports SQLAlchemy

app = Flask(__name__)
app.config['SQLALCHEMY_DATABASE_URI'] = 'sqlite:///example.db' //This line tells the Flask application to use SQLAlchemy to connect to an SQLite database stored in a file named example.db. This database file will be created in the root directory of your project if it doesn't already exist.
db = SQLAlchemy(app) //SQLAlchemy(app): This initializes the SQLAlchemy instance with the Flask application instance app.

Flask-WTF

Flask-WTF is an extension for Flask that adds support for WTForms, a flexible forms validation and rendering library for Python web development.

from flask_wtf import FlaskForm
from wtforms import StringField, SubmitField
from wtforms.validators import DataRequired

//Im this form example, we have to things, a name input and a submt button
class MyForm(FlaskForm):
    name = StringField('Name', validators=[DataRequired()]) //This is the variable name for the form field. It is used to reference this field within your Flask application.
    submit = SubmitField('Submit') //This is the variable name for the submit button. It is used to reference this button within your Flask application.

Flask-Login

Flask-Login is an extension for Flask that adds support for user authentication, including session management and login/logout functionality.

from flask_login import UserMixin

class User(UserMixin): //is a class that provides default implementations for the methods that Flask-Login expects user objects to have, such as is_authenticated, is_active, is_anonymous, and get_id
    def __init__(self, id): //This is the constructor method for the User class. It initializes a new instance of the User class with an id attribute
        self.id = id

from flask_login import LoginManager //LoginManager is the main class used to manage user login sessions in a Flask application.

login_manager = LoginManager() // This instance will be used to configure and manage user login sessions.
login_manager.init_app(app) // It tells Flask-Login to use this instance to manage user sessions for your application.

from flask_login import login_user, logout_user, login_required, current_user
