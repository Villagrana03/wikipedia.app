Handling Static Files

Handling static files in Flask involves managing files like CSS, JavaScript, and images that your web application uses to create a richer user experience.

Serving Static Files (CSS, JavaScript, Images)

Static files are assets that don’t change dynamically during the application runtime, such as stylesheets, JavaScript scripts, images, or fonts.

  • Flask provides a built-in way to serve these files using a folder named static in your project directory.
  • When you want to include these files in your application, Flask allows you to link them directly in your HTML templates using a special helper function called url_for.

The static Folder and Flask Configuration

/project
├── app.py
├── templates/
└── static/
--- ├── css/
--- │ └── styles.css
--- ├── js/
--- │ └── script.js
--- └── images/
--- └── logo.png

Accessing Files

Flask automatically serves files from the static folder when you use the URL example /static/< filename >

Customizing Static Folder: If you want to use a different folder name, you can configure Flask like this:

app = Flask(__name__, static_folder='assets')

Linking Static Files in Templates

To use static files in your HTML templates, Flask provides the url_for() function. This function generates the correct URL to access files in the static folder.

//In this example we have the folder named "static"
< body >
--- < img src="{{ url_for('static', filename='images/logo.png') }}" alt="Logo" >
< /body >
