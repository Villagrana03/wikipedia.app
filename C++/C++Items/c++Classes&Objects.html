Classes & Objects

Get started with object-oriented programming.

Class

A class is a user-defined data type that encapsulates information and serves as a blueprint for objects, which are instances of the class.

To create a class, use the class keyword in the following syntax:

class School {

}; // ← Semicolon is required

Class members

A member of a class is any variable or method defined within that class. Members define the properties (data) and behaviors (functions) of objects created from that class.

  1. Member Variables: These are the attributes or properties of the class. They hold the state or data of an object. For example, in a Car class, color and model could be member variables.
  2. Member Methods: These are functions that define the behavior of the class. They can operate on member variables or perform specific actions related to the object. For instance, a method like startEngine() in a Car class would control the car's behavior

We will extent School class!

//Here we added 3 members, 2 variables and 1 method.
class School {
public:
--- std::string name;
--- int age;

void banner() {
--- std::cout << name << " is " << age << " years old.\n";
}
};

Defining Methods Outside of a Class

For larger projects, it’s recommended to split up the class definition in a header and a .cpp file. Doing so requires class methods to be defined outside of the class definition. In this scenario, use the ClassName:: namespace to indicate that the function belongs to that class.

//In this example i am declaring the method outside of the casss ("School")
void School::banner() {
--- std::cout << name << " is " << age << " years old.\n";
}

//Note: Even though a method can be defined outside its class, it must still be declared inside it.`

Objects

An object is a specific instance of a class in object-oriented programming. It represents a distinct entity that combines both data (attributes) and behaviors (methods) defined by the class.

Create a object from School Class

School codecademy;

Once an object is created, we can access and modify the class attributes using the dot operator .:

//// Set values for attributes name and age
codecademy.name = "Codecademy";
codecademy.age = 10;

//Here we are accesing a class method trough the object!
codecademy.banner();

Multiple Objects

// Create an object of School
School harvard;
harvard.name = "Harvard";
harvard.age = 385;

// Create another object of School
School yale;
yale.name = "Yale";
yale.age = 319;

Access Specifiers

Access specifiers are C++ keywords that determine the scope of class components. This article will cover the following topics:

  • The private keyword
  • The public keyword
  • Encapsulation
  • Accessor functions
  • Mutator functions

private keyboard

a private class typically refers to a class that is declared as a member of another class and has its access level set to private. This means that the private class cannot be accessed directly from outside the outer class.

NOTE: we will use private classes ONlY when is going to be a sub class of a class

Reasons to use a private class
  • Encapsulation: A private class encapsulates its implementation details, meaning that external code cannot directly access or modify its members. This protects the integrity of the data and behavior.
  • Controlled Access: By limiting access to the class, you ensure that only certain parts of your program can create or manipulate instances of that class, reducing the risk of misuse.
  • Protection from External Changes:A private class can evolve independently from external code. This isolation can prevent bugs and unintended side effects from changing internal logic.
  • Easier Maintenance: Changes to the implementation of a private class do not affect external code, as long as the public interface remains unchanged. This makes the code easier to maintain and refactor.

class Student {
std::string name;
};

int main() {
Student clara;
clara.name = "Clara"; // Not allowed, name is private
return 0;
}

Public Keyboard

The public access specifier has the opposite effect of private; public class members are accessible from anywhere in the program.

class Festival {
public:
int firework;
};

int main() {
Festival christmas;
christmas.firework = 100; // Allowed because firework is public
}

Encapsulation

Encapsulation is the concept of bundling the data (attributes) and the methods (functions) that operate on that data into a single unit, called a class. It restricts direct access to some of the object's components, which is a means of preventing accidental interference and misuse of the methods and data.

In C++, encapsulation is achieved by declaring class members/attributes as private so they cannot be accessed from outside of the class.

Accessor Functions

An accessor function (also known as a “getter” function) is a public function that returns the value of a private member variable:

class Clock {
private:
int time = 1200;

public: // Accessor function for time
int getTime() {
return time;
}
};

int main() {
//The main() function calls the accessor function getTime() and gets the value of the time attribute even though it is private.
Clock alarm;
std::cout << alarm.getTime(); //Output: 1200
}

Mutator Functions

class Clock {
private:
int time = 1200;

public:
// Accessor function for time
int getTime() {
return time;
}

// Mutator function for time
void setTime(int new_time) {
time = new_time;
}
};

int main() {
Clock alarm;
alarm.setTime(930);
std::cout << alarm.getTime(); // Output: 930 }

Mutator functions usually have a return type of void. They are only responsible for resetting the value of an existing class member. As such, mutator functions often have one parameter of the same type as the attribute.
